# AGENTS.md

## 作用域

本文件约束整个 `github.com/octohelm/objectkind` 仓库。子目录若存在更近的 `AGENTS.md`，其明确重写的规则优先；未重写的规则继续继承本文件。

只有落盘到 `AGENTS.md` 的规则才作为仓库长期协作规则生效。对话中的临时偏好只在当前任务内生效，不应被写成长期约束，除非用户明确要求。

## 起手动作

1. 改动前先做最小盘点：确认目标路径、现有入口、相关测试和工作区脏改。
2. 优先阅读当前任务直接涉及的源码、`go.mod`、`justfile` 和相关生成入口。
3. 发现未提交变更时，先判断是否与任务相关；不得回滚、覆盖或整理自己未引入的变更。

## 变更边界

1. 只修改与当前任务直接相关的仓库内文件。
2. 不做顺手重构、批量格式化、依赖升级或入口重排，除非它们是完成任务的必要部分。
3. 不因目录存在而默认新增模块、workspace、执行入口或注册关系。
4. 不直接编辑 `zz_generated.*` 等生成文件，除非任务明确要求或已经运行对应生成入口。
5. 高风险操作、破坏性 git 操作、发布流程和会影响外部环境的命令必须先得到明确确认。

## 架构边界

1. `tool/internal/cmd/gen` / `tool/internal/cmd/fmt` / `tool/internal/cmd/skills-install` 承载仓库开发工具 CLI 入口；`internal/example/cmd/example` 承载示例运行入口装配；业务能力优先沉到 `pkg/*` 或 `internal/*`。
2. `pkg/*` 面向复用能力与公共接口；`internal/*` 面向仓库内部实现和开发工具。
3. `tool/go/justfile` 是 Go 工具链入口；根 `justfile` 只聚合稳定仓库入口和运行入口。
4. `tool/` 和 `.agents/skills/` 属于工具链与技能相关控制面，修改时必须同步检查对应执行入口。

## 技能与生成约束

1. `go.mod` 中带 `+skill:*` 注释的依赖表示相关任务应优先使用对应 skill。
2. 涉及 `+gengo`、代码生成、枚举或生成入口时，先读取对应 skill 或本地生成器实现，再改注解或生成配置。
3. 修改测试时优先遵循仓库已使用的 `github.com/octohelm/x/testing/v2` 风格。

## 验证义务

1. 能完成最小相关验证时，必须先验证再交付。
2. Go 代码改动优先运行相关包测试；跨包或公共接口改动再运行 `just go test` 或等价 `go test ./...`。
3. 修改生成配置或生成器时，运行对应 `go generate`、`just go gen` 或明确说明未运行原因。
4. 修改 `justfile` 时，至少运行 `just --list --list-submodules` 或对应 `just --usage <recipe>`。
5. 无法验证时，交付中必须说明缺口、原因与风险。

## 暂停条件

以下情况必须先暂停并请求确认：

1. 需求边界不清且存在多种互斥实现。
2. 需要新增模块、workspace、执行入口、发布入口或长期注册关系。
3. 需要改变根 `justfile` 的入口分层、默认行为或 CI 调用方式。
4. 需要回滚、覆盖或重写用户已有变更。
5. 发现项目结构、控制面职责或流水线入口存在异常，且继续落地会扩大影响面。

## 表达要求

1. 尽量使用中文，除非专用名称、API 名称或已有英文上下文更合适。
2. 注释、错误和日志应直接、准确、可验证。
3. 交付说明只陈述已完成并已验证的事实；未验证项、假设和风险必须显式区分。
